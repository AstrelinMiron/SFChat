3. Описание выбранной идеи решения:
Проект обеспечивает базовую функциональность. При работе приложения "залогиненным" может быть только один пользователь.
Чат имеет основное меню для выполнения входа или регистрации, а также подменю для вошедших пользователей.

Регистрация пользователей выполняется по логину (email), имени и паролю.
Вход выполняется по логину (email) и паролю. 
После входа пользовательское меню предлагает на выбор следующие действия: 
- вывести историю сообщений;
- отправить сообщение всем зарегестрированным пользователям или конкретному, указв имя получателя;
- вывести список зарегестрированных пользователей;
- выйти в основное меню.
4. Описание пользовательских типов и функций в проекте: 
Реализация класса Chat использует контейнеры "vector" для хранения объектов типа User и Message.
В программе присутствует группа функций для запуска и отображения меню чата(startChat(), showLoginMenu(), howUserMenu()), 
а также группа функий реализующая предлагаемые действия, указанные в меню: 
- Вход в чат по логину и паролю - userLogin();
- Регистрация в чате - userRegistration();
- Возвращает историю сообщений - showChat();
- Возвращает список пользователей - showAllUsers();
- Отправка сообщения - addMessage();
